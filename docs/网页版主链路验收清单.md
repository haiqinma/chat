# 网页版主链路验收清单

本文档用于验证 Chat standalone 网页版从登录到使用 Skill 的核心流程。它是当前阶段的人工回归基线，不要求一次性建设完整的浏览器自动化测试。

每次修改登录、账户切换、同步、Router、会话初始化或发现页后，都应至少回归本清单中的 P0 项。

## 1. 验收前提

- 使用本地 standalone 开发服务访问 `http://localhost:3020`。
- 准备至少两个已经使用过的账户，并确保账户 A、账户 B 的会话和配置不同。
- Router 至少有一个可用文本模型。
- 准备一个没有可用模型的账户或测试配置，用于验证配置引导。
- 网页版不依赖桌面端能力，不验证本地文件、Git 或本地 Tool Runtime。

## 2. P0 主链路

| 编号  | 场景                   | 预期结果                                               |
| ----- | ---------------------- | ------------------------------------------------------ |
| P0-01 | 首次打开未登录页面     | 页面稳定停留在登录页，不出现空会话或模型配置页         |
| P0-02 | 钱包签名成功           | 进入同步状态页或明确 loading，不短暂展示空白会话       |
| P0-03 | 同步完成且 Router 可用 | 进入可使用页面，左侧会话和配置完整恢复                 |
| P0-04 | 同步失败               | 保持登录状态，显示可理解的错误和重试入口，不回到登录页 |
| P0-05 | Router 没有可用模型    | 进入模型配置引导，不创建带默认模型的空通用会话         |
| P0-06 | 刷新已登录页面         | 保持当前账户、会话、Router 配置和 Skill 状态           |
| P0-07 | 账户 A 切换到账户 B    | 显示同步 loading，最终只显示账户 B 数据                |
| P0-08 | 账户 B 切回账户 A      | 恢复账户 A 原有会话和配置，不创建重复通用会话          |
| P0-09 | 新建通用会话           | 默认是简单问答，不启用深度思考、不发送 tools           |
| P0-10 | 发送通用会话消息       | 请求只包含当前模型和消息，不包含 Skill 专属 tools      |
| P0-11 | 发现页启用 Skill       | “配置并启用”打开配置，保存后 Skill 变为可使用          |
| P0-12 | 使用已启用 Skill       | 进入该 Skill 的会话或工作区，并使用其模型和工具绑定    |

## 3. P1 回归项

| 编号  | 场景                    | 预期结果                                               |
| ----- | ----------------------- | ------------------------------------------------------ |
| P1-01 | 锁定钱包后重新打开 Chat | 登录状态失效时回到登录流程，不进入错误配置页           |
| P1-02 | 钱包切换账户            | Chat 能感知账户变化并切换对应工作区                    |
| P1-03 | 登录后等待一小时以上    | 不因前端状态丢失而自动退出；凭证失效时给出明确原因     |
| P1-04 | Router 模型刷新         | 返回模型列表时正确展示 tags、端点和可用状态            |
| P1-05 | 只购买图片模型          | 新建会话仍可进入 AI 绘画，不把文本会话错误引导到购买页 |
| P1-06 | Skill 缺少依赖          | 显示缺少的模型或 Tool，不出现点击无反应                |
| P1-07 | 桌面版打开同一账户      | 核心账户、Router、会话和 Skill 数据可按产品边界恢复    |

## 4. 每次修复后的最小回归

如果改动属于登录、账户或同步，至少执行：

- P0-01 至 P0-08
- P1-01 至 P1-03

如果改动属于模型、会话或 Tool，至少执行：

- P0-05、P0-09、P0-10
- P1-04 至 P1-06

如果改动属于 Skill 或发现页，至少执行：

- P0-03、P0-05、P0-09、P0-11、P0-12
- P1-06
